# Bimanual Robot Dataset Visualizer

A comprehensive GUI tool for visualizing dual-arm robot demonstration datasets with synchronized multi-modal data display.

## Features

### ✨ Core Capabilities

- **Dual-Arm Pose Visualization**
  - Numerical display of position (X, Y, Z) and rotation (axis-angle)
  - 3D coordinate frame visualization with XYZ axes
  - Real-time synchronized updates

- **Gripper State Display**
  - Gripper width (opening) for both arms
  - Numerical values in meters

- **Multi-Camera Views**
  - Synchronized display of all camera feeds
  - Automatic layout for multiple cameras
  - RGB image visualization

- **Playback Controls**
  - Episode navigation (previous/next)
  - Play/Pause functionality
  - Adjustable playback speed (1-100 FPS)
  - Frame-by-frame scrubbing with slider
  - Progress indicator

- **Time Synchronization**
  - All modalities (poses, grippers, cameras) are perfectly time-aligned
  - Single frame index controls all visualizations

### GUI Controls

#### Episode Navigation
- **Episode Selector**: Dropdown to select specific episode (0 to N-1)
- **◀◀ Prev Episode**: Jump to previous episode
- **Next Episode ▶▶**: Jump to next episode

#### Playback Controls
- **▶ Play / ⏸ Pause**: Start/stop automatic playback
- **Frame Slider**: Scrub through frames manually
- **Speed Slider**: Adjust playback speed (1-100 FPS)

#### Display Panels

**Left Panel - 3D Visualization**
- Interactive 3D view of robot end-effector poses
- Red coordinate frame: Robot 0 (right arm)
- Blue coordinate frame: Robot 1 (left arm)
- X-axis: Red, Y-axis: Green, Z-axis: Blue
- Mouse controls:
  - Left drag: Rotate view
  - Right drag: Pan view
  - Scroll: Zoom in/out

**Middle Panel - Robot States**
- **Robot 0 State**: Position, rotation, gripper width
- **Robot 1 State**: Position, rotation, gripper width
- All values in meters and radians

**Bottom Panel - Camera Views**
- Camera 0, Camera 1, ... Camera N
- Synchronized RGB images
- Auto-scaled to fit display

## Data Format

The visualizer expects a zarr dataset with the following structure:

```
dataset.zarr.zip
├── data/
│   ├── robot0_eef_pos          # (T, 3) - Robot 0 end-effector position
│   ├── robot0_eef_rot_axis_angle  # (T, 3) - Robot 0 rotation (axis-angle)
│   ├── robot0_gripper_width    # (T, 1) - Robot 0 gripper opening
│   ├── robot1_eef_pos          # (T, 3) - Robot 1 end-effector position
│   ├── robot1_eef_rot_axis_angle  # (T, 3) - Robot 1 rotation (axis-angle)
│   ├── robot1_gripper_width    # (T, 1) - Robot 1 gripper opening
│   ├── camera0_rgb             # (T, H, W, 3) - Camera 0 images
│   ├── camera1_rgb             # (T, H, W, 3) - Camera 1 images
│   └── ...
└── meta/
    └── episode_ends            # Episode boundary indices
```

This format is automatically generated by `scripts_slam_pipeline/07_generate_replay_buffer.py`.

## Troubleshooting

### Issue: "No module named 'PyQt5'"
```bash
pip install PyQt5
```

### Issue: "No module named 'OpenGL'"
```bash
pip install PyOpenGL
```

### Issue: 3D view not rendering
- Ensure you have OpenGL support on your system
- Update graphics drivers
- Try running with software rendering:
  ```bash
  export LIBGL_ALWAYS_SOFTWARE=1
  python visualize_dataset.py <dataset>
  ```

### Issue: Images not displaying
- Check that dataset contains `camera*_rgb` arrays
- Verify image data is in uint8 RGB format
- Check dataset was generated correctly

### Issue: Slow playback
- Reduce playback speed
- Close other applications
- Dataset may be compressed - first load is slower

## Technical Details

### Time Synchronization
All data modalities share the same time index:
- Frame index `i` corresponds to the same timestamp across all sensors
- Episode boundaries are defined by `meta/episode_ends`
- Global frame index = `episode_start + frame_offset`

### Coordinate Systems
- **Position**: Meters (X, Y, Z)
- **Rotation**: Axis-angle representation (radians)
  - Rotation vector direction = axis of rotation
  - Rotation vector magnitude = angle of rotation
- **3D Visualization**: Right-handed coordinate system
  - X: Red (forward)
  - Y: Green (left)
  - Z: Blue (up)

### Performance
- Lazy loading: Only current frame is loaded into memory
- Compressed storage: Images stored with JPEG-XL compression
- Efficient rendering: PyQtGraph uses OpenGL acceleration

### Export frames as images
Modify the script to save frames:
```python
# Add to update_visualization() method
import cv2
for i, panel in enumerate(self.camera_panels):
    image = frame_data[f'camera{i}_rgb']
    cv2.imwrite(f'frame_{self.current_frame}_cam{i}.png', 
                cv2.cvtColor(image, cv2.COLOR_RGB2BGR))
```

## Development

### Adding New Visualizations

To add custom visualizations, modify `visualize_dataset.py`:

1. **Add new panel class**:
```python
class CustomPanel(QWidget):
    def __init__(self):
        super().__init__()
        # Setup UI
    
    def update_data(self, data):
        # Update visualization
        pass
```

2. **Integrate in main window**:
```python
# In setup_ui()
self.custom_panel = CustomPanel()
main_layout.addWidget(self.custom_panel)

# In update_visualization()
self.custom_panel.update_data(frame_data)
```

### Extending Data Support

To support additional data types:
1. Modify `DatasetLoader.get_frame_data()` to load new data
2. Create visualization panel for new data type
3. Add to main window layout
4. Update in `update_visualization()`
